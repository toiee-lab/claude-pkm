# Claude PKM (Personal Knowledge Management)

Claude Project機能を最大限に活用するためのドキュメント管理システムです。ベストプラクティスに従ったドキュメント作成・管理により、Claudeがより効果的に情報を理解・活用できる環境を構築します。

## 特徴

- **自然言語でドキュメント管理**: 「プロファイルドキュメントを作成して」といった自然な指示で操作
- **品質保証**: 5つの観点（完全性・正確性・明確性・一貫性・Claude最適化）から自動レビュー
- **最適化機能**: Claude Projectに最適化されたドキュメント構造への自動変換
- **明確なガイドライン**: ファイル命名規則、ディレクトリ構造、ドキュメント構造のルールを提供

## ディレクトリ構造

```
claude-pkm/
├── profile/          # 個人プロファイル（自己紹介、作業スタイル、コミュニケーション設定）
├── projects/         # プロジェクト別のコンテキスト
├── knowledge/        # ナレッジベース（技術知識、ドメイン知識）
└── instructions/     # Claudeへの指示（一般指示、タスク別指示）
```

## クイックスタート

### 1. 新規ドキュメント作成

```
「プロファイルのドキュメントを作成して」
```

Claudeがカテゴリを判定し、適切な場所にファイルを作成します。

### 2. ドキュメントレビュー

```
「このドキュメントをレビューして」
```

50点満点でドキュメントの品質を評価し、改善提案を提供します。

### 3. ドキュメント最適化

```
「ドキュメントを最適化して」
```

Claude Projectに最適な構造へ自動的に改善します。

## 主要機能

### ドキュメント作成

- **カテゴリ自動判定**: profile/project/knowledge/instructionから適切なカテゴリを選択
- **ファイル名提案**: 小文字ケバブケース形式で既存ファイルと重複しない名前を提案
- **テンプレート適用**: ベストプラクティスに従った構造で作成

### ドキュメントレビュー

5つの観点から評価（各10点満点、合計50点）:

- **完全性**: 必須要素の存在確認
- **正確性**: 情報の正確さと具体性
- **明確性**: 構造の明確さと可読性
- **一貫性**: 命名規則とスタイルの統一
- **Claude最適化**: Claudeの理解しやすさ

### ドキュメント最適化

- **構造チェック**: 見出し階層、セクション分割の適切性
- **可読性チェック**: 段落の長さ、リスト・表の使用
- **Claude最適化チェック**: 相対参照の排除、自己完結性の確保

## ドキュメント規則

### ファイル命名規則

- 形式: 小文字ケバブケース（例: `my-document.md`）
- 言語: 英語のみ
- 拡張子: `.md`（Markdown）

### ドキュメント構造

1. H1タイトルで開始
2. 見出し階層を順守（H1→H2→H3、スキップしない）
3. タイトル直後に概要を記述
4. 適切な見出しでセクションを分割

### ベストプラクティス

- **明確なファイル名**: ファイル名から内容を推測できるように
- **包括的なコンテンツ**: 関連情報をまとめて提供
- **関連コンテンツの整理**: 関連ドキュメントを同じプロジェクトに配置
- **具体的な参照**: ドキュメント名を明示して参照

## Claude Projectへの取り込み

以下のファイルを取り込むことを推奨：

- `profile/` 内の全ファイル
- `projects/` から関連するファイル
- `instructions/general-instructions.md`
- 必要な `knowledge/` 内のファイル
